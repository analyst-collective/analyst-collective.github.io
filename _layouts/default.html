The Idea Behind Analyst Collective

There's never been a better time to be an analyst.

Underlying data technology has improved to a point where interactive query response time is no longer The Big Problem. A myriad of tools have been built on top of this technology that provide a wide array of options for analysts looking to analyze and visualize data.

But the core analytical workflow is still lacking core features that it needs. Analysts donâ€™t do a good job of testing, using version control, documentation, designing for maintainability, or collaborating with other analysts. This leads to poor outcomes, both for analysts and for the businesses that employ them.

The Analyst Collective viewpoint is that analytic practices should be modeled off of software engineering. Specifically:

  • Analytics should be collaborative
  • Analytic code is an asset
  • Analytics requires automation
